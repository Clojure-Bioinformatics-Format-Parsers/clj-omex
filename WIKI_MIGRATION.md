# Wiki Migration Instructions

## Content to Move to Repository Wiki

The following content from the `/docs` folder should be manually moved to the repository wiki:

### DESIGN.md

The file `/docs/DESIGN.md` contains comprehensive design documentation for clj-omex that should be added to the repository wiki. This includes:

- Architecture overview
- Namespace organization
- Core design principles
- Structured error model
- RDF extraction patterns
- URI normalization
- Protection mechanisms
- SPARQL support
- Statistics model
- Testing strategy
- Future enhancements
- Dependencies

**Action Required**: Create a wiki page titled "Design" and copy the content from `/docs/DESIGN.md` to that page.

## Wiki Setup

If the wiki doesn't exist yet:

1. Go to the repository Settings
2. Enable the Wiki feature
3. Create the wiki pages as needed

After the content has been migrated to the wiki, this file can be deleted.
